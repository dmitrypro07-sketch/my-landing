## 1. Цель и аудитория

### Главная цель сайта
Конвертировать холодный трафик (реклама, рекомендации, органика) в заявку через WhatsApp или Telegram на бесплатный разбор проекта.

**Вторичные цели:**
- Сформировать доверие через кейсы и демо ещё до контакта
- Отсеять нецелевых клиентов (нет бюджета / не подходит ниша)
- Позиционировать специалиста как эксперта, а не «сделаю дёшево»

### Целевая аудитория

**Первичная (горячая):**
- Владельцы малого/среднего бизнеса, 28–50 лет
- Оборот: 500 000 — 5 000 000 ₽/мес
- Ниши: медицина/стоматология, недвижимость, онлайн-образование, ритейл/e-com, юридические услуги, салоны красоты
- Боль: теряют лиды, менеджеры не успевают, нет ответа ночью/в выходные
- Устройство: 65–70% мобильный (iPhone, Android)
- Трафик: Instagram, ВКонтакте, Яндекс.Директ, Telegram-каналы, сарафан

**Вторичная (тёплая):**
- Маркетологи и операционные директора, которые ищут подрядчика для внедрения
- Предприниматели, уже слышавшие об ИИ-ботах, но не знающие как это работает

### Ключевые страхи аудитории
1. «Потрачу деньги — результата не будет»
2. «Бот будет отвечать невпопад и отпугнёт клиентов»
3. «Слишком сложно, я не разберусь»
4. «Это дорого» (не знают цен)
5. «А это вообще законно / безопасно для данных?»

### Главное возражение
«Я не уверен, что это подходит именно для моего бизнеса»

## 2. Технический стек

```
Разметка:     HTML5, семантические теги (section, article, nav, main)
Стили:        CSS3, custom properties, Grid, Flexbox, clamp()
Скрипты:      Vanilla JS (~200 строк), без фреймворков
Шрифты:       Google Fonts — Golos Text + Inter + JetBrains Mono
Анимации:     Intersection Observer + CSS transitions + GSAP (только для hero)
Smooth scroll: Lenis (~3.5kb)
Иконки:       Lucide Icons (SVG sprite, MIT license)
Форма:        Нативная HTML-форма → webhook (или прямая ссылка WhatsApp)
Хостинг:      Timeweb / Beget / Selectel (РФ датацентр) + Cloudflare CDN
Сборка:       Без сборщика (чистый HTML/CSS/JS) или Vite для удобства
```

**Целевые метрики производительности:**
- LCP < 2.5 сек на мобильном (4G)
- FCP < 1.2 сек
- CLS < 0.1
- Размер страницы < 1.5 MB
- PageSpeed Score ≥ 90 (мобильный)

### 01. HERO — Первый экран

**Цель:** Ответить «Это для меня?» за 3 секунды. Один CTA. Без скролла — всё нужное видно.

**Макет:** Split-layout (50/50)
- Левая половина: текст + кнопки
- Правая половина: живая демо-витрина или анимация диалога

**Фон:** Mesh gradient (`--gradient-glow-1` + `--gradient-glow-2`) + dot grid overlay (3% opacity) + noise texture (4% opacity)

---

**Контент (левая часть):**

```
[Badge-pill: «ИИ-продавец для переписок»]

ХЕДЛАЙН:
Ваши клиенты пишут ночью.
Кто им отвечает?

СУБ-ХЕДЛАЙН:
Создаю ИИ-ассистентов, которые ведут переписки,
квалифицируют лидов и записывают на встречу 24/7 —
в WhatsApp, Telegram, Instagram и на сайте.

[Кнопка PRIMARY: «Написать в WhatsApp →»]
[Кнопка SECONDARY: «Посмотреть как работает ↓»]

[Micro social proof:]
★★★★★  Уже запущено 47+ ботов  |  Среднее время ответа: 18 сек
```

**Контент (правая часть) — демо-витрина:**

Анимированный интерфейс мессенджера (имитация WhatsApp / Telegram), в котором разворачивается диалог:

```
👤 Клиент: Здравствуйте, хочу записаться на консультацию

🤖 ИИ-ассистент: Добрый день! С удовольствием помогу записаться.
   Как вас зовут и на какой день вам удобнее?

👤 Клиент: Анна, хотелось бы на среду

🤖 ИИ-ассистент: Анна, у нас есть свободное время в среду:
   • 11:00
   • 14:30
   • 17:00
   Какое подходит?

👤 Клиент: 14:30

🤖 ИИ-ассистент: ✅ Готово! Записала вас на среду в 14:30.
   Напомню за час до встречи.
```

Сообщения появляются последовательно с задержкой (typing indicator между ответами). Цикл повторяется каждые 8 секунд.

**Анимации hero:**
- Хедлайн: word-split animation (слова вылетают снизу с stagger 80ms)
- Суб-хедлайн: fadeInUp с задержкой 400ms
- Кнопки: fadeInUp с задержкой 600ms
- Демо-витрина: появляется справа с задержкой 200ms
- Badge: pop-in с cubic-bezier пружиной

## 6. Анимации и интерактив

### Обязательные анимации

| Элемент | Анимация | Библиотека |
|---|---|---|
| Hero хедлайн | Word split, stagger 80ms | Vanilla JS + CSS |
| Hero суб-хедлайн | fadeInUp, delay 400ms | CSS |
| Hero кнопки | fadeInUp, delay 600ms | CSS |
| Демо-витрина | Последовательное появление сообщений | Vanilla JS |
| Счётчики (proof bar) | animateCounter при входе в вьюпорт | Vanilla JS |
| Все секции | fadeInUp при входе (Intersection Observer) | Vanilla JS |
| Stagger карточки | Каждая карточка с delay += 80ms | Vanilla JS |
| Primary CTA кнопка | Pulsing glow keyframe | CSS |
| Nav при скролле | Glass transition | CSS + JS |
| Счётчик слотов | Печатающийся текст или статичный | CSS |

### Hover-эффекты

| Элемент | Hover |
|---|---|
| Карточки услуг | Spotlight cursor tracking gradient |
| Карточки кейсов | Glassmorphism усиление + border glow |
| Primary Button | translateY(-2px) + усиление glow |
| Ссылки навигации | Underline slide из левого края |
| FAQ строки | Цвет текста → accent, левая border |
| Social links | Scale 1.1 + color accent |

### Дополнительно (nice to have)

- **Lenis smooth scroll** — обязательно для premium-ощущения
- **Typing indicator** в демо-витрине (3 точки перед ответом бота)
- **Border beam** на выделенном тарифе «Бизнес»
- **Scramble text** для счётчиков при первом появлении

### Что НЕ использовать
- Тяжёлые particle-системы (убивают мобильную производительность)
- Autoplay видео со звуком
- GSAP для простых reveal-анимаций (замена: CSS + Intersection Observer)
- Горизонтальный scroll на мобильном
